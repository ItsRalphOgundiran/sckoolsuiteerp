☁️ Database Hosting Options

Option 1: Railway RECOMMENDED

Best for: Quick setup, small to medium schools

Pricing: Free tier available, then ~$5-20/month

Difficulty: Beginner-friendly

  • One-click PostgreSQL provisioning
  • Automatic backups
  • Connection pooling included
  • Deploy via Git push

Option 2: Supabase ENTERPRISE

Best for: Large schools, high compliance requirements

Pricing: Generous free tier, then ~$25+/month

Difficulty: Intermediate

  • Enterprise-grade PostgreSQL
  • Built-in connection pooling
  • Row-level security
  • Excellent for scaling

Option 3: Self-Hosted VPS ADVANCED

Best for: Data sovereignty, full control

Pricing: $5-20/month for VPS

Difficulty: Advanced (requires Linux knowledge)

  • Complete data ownership
  • Full configuration control
  • Requires manual backup setup

🚂 Option 1: Railway Setup (Step-by-Step)

Step 1: Create Railway Account

  1. Visit https://railway.app
  2. Click "Start for Free"
  3. Sign up using GitHub (recommended) or email
  4. Verify your email address

Step 2: Create New Project

  1. Click "New Project"
  2. Select "Provision PostgreSQL"
  3. Wait 1-2 minutes for database to provision
  4. Click on the PostgreSQL service card

Step 3: Get Database Connection String

  1. In PostgreSQL service, click "Connect" tab
  2. Look for "DATABASE_URL"
  3. Click copy button (or click the value to reveal)
  4. Save this somewhere secure - you'll need it!
✅ Example Railway DATABASE_URL: 
postgresql://postgres:AbCdEfGhIjKlMnOp@containers.railway.app:5432/railway

Step 4: Configure Environment Variables

Create a .env.local file in your project root:

# Database (Railway PostgreSQL)
DATABASE_URL="postgresql://username:password@containers.railway.app:5432/railway"
DIRECT_URL="postgresql://username:password@containers.railway.app:5432/railway"

# NextAuth Configuration
NEXTAUTH_URL="http://localhost:3000"
NEXTAUTH_SECRET="your-random-secret-min-32-characters-long"

# Node Environment
NODE_ENV="development"
🔐 Generate NEXTAUTH_SECRET: 
openssl rand -base64 32
Or use an online generator. Must be at least 32 characters.

Step 5: Install Dependencies

cd /path/to/sckoolsuiteerp
npm install

# Generate Prisma client for PostgreSQL
npx prisma generate

# Run database migrations
npx prisma migrate deploy

Step 6: Start Development Server

npm run dev

Visit http://localhost:3000/setup to create your school.

Step 7: Deploy to Railway

# Install Railway CLI
npm install -g @railway/cli

# Login to Railway
railway login

# Link your project
railway link

# Deploy!
railway up

🔷 Option 2: Supabase Setup

Step 1: Create Supabase Project

  1. Visit https://supabase.com
  2. Sign in with GitHub
  3. Click "New Project"
  4. Choose organization and project name
  5. Select region closest to your users
  6. Set a secure database password (save this!)
  7. Wait for project creation (2-3 minutes)

Step 2: Get Connection Strings

  1. In your project dashboard, click "Connect"
  2. Go to "ORMs" tab
  3. Find "Prisma" section
  4. Copy both URLs:
Connection Settings for Supabase: 
# Transaction pooler (for queries)
DATABASE_URL="postgresql://postgres.[ref]:[password]@aws-0-[region].pooler.supabase.com:6543/postgres?pgbouncer=true"

# Direct connection (for migrations)
DIRECT_URL="postgresql://postgres:[password]@db.[ref].supabase.co:5432/postgres"

Step 3: Configure Environment

# .env.local
DATABASE_URL="your-supabase-pooler-url"
DIRECT_URL="your-supabase-direct-url"
NEXTAUTH_URL="http://localhost:3000"
NEXTAUTH_SECRET="your-generated-secret"
NODE_ENV="development"

Step 4: Deploy Migrations

npx prisma migrate deploy

🖥️ Option 3: Self-Hosted VPS Setup

Requirements

  • Ubuntu 22.04 LTS server
  • 2+ CPU cores
  • 4GB RAM minimum (8GB recommended)
  • 50GB SSD storage

Installation Commands

# Update system
sudo apt update && sudo apt upgrade -y

# Install PostgreSQL
sudo apt install postgresql postgresql-contrib -y

# Start and enable PostgreSQL
sudo systemctl start postgresql
sudo systemctl enable postgresql

# Create database and user
sudo -u postgres psql -c "CREATE DATABASE sckoolsuite;"
sudo -u postgres psql -c "CREATE USER sckooladmin WITH ENCRYPTED PASSWORD 'your-secure-password';"
sudo -u postgres psql -c "GRANT ALL PRIVILEGES ON DATABASE sckoolsuite TO sckooladmin;"
sudo -u postgres psql -c "ALTER USER sckooladmin WITH SUPERUSER;"

# Configure PostgreSQL for remote connections
sudo nano /etc/postgresql/14/main/postgresql.conf
# Set: listen_addresses = '*'

sudo nano /etc/postgresql/14/main/pg_hba.conf
# Add: host all all 0.0.0.0/0 md5

sudo systemctl restart postgresql

# Open firewall
sudo ufw allow 5432/tcp
sudo ufw reload

🚀 Application Deployment

Deploy to Vercel (Recommended)

# Install Vercel CLI
npm i -g vercel

# Login
vercel login

# Initialize project
vercel

# Add environment variables in Vercel Dashboard:
# 1. Go to Project Settings → Environment Variables
# 2. Add DATABASE_URL
# 3. Add DIRECT_URL (if using Supabase)
# 4. Add NEXTAUTH_SECRET
# 5. Add NEXTAUTH_URL (your production domain)

# Deploy to production
vercel --prod

Environment Variables Reference

Variable Required Description
DATABASE_URL Yes PostgreSQL connection string
DIRECT_URL Sometimes Direct connection for migrations (Supabase)
NEXTAUTH_SECRET Yes 32+ character random string
NEXTAUTH_URL Yes Your production domain
NODE_ENV Yes "production" for live sites

🔒 Security Checklist

Before going live, verify:
  • ☐ Database not publicly accessible (use private networking)
  • ☐ Strong database password (32+ random characters)
  • ☐ NEXTAUTH_SECRET is cryptographically random
  • ☐ Environment variables set in hosting platform (not in code)
  • ☐ HTTPS enabled on production domain
  • ☐ Database backups configured (daily automated)
  • ☐ .env files added to .gitignore
  • ☐ Two-factor authentication enabled on hosting accounts

🔧 Troubleshooting

"Can't reach database server" Error

# Verify connection string format
postgresql://username:password@hostname:5432/database

# Test connection with psql
psql "postgresql://username:password@hostname:5432/database"

# Check if IP is allowlisted in database settings

Migration Failures

# Reset and redeploy
npx prisma migrate reset --force
npx prisma migrate deploy

# If still failing, check logs
npx prisma migrate status

Connection Pool Exhausted

With Supabase/Railway, connection pooling is handled automatically. For self-hosted, install PgBouncer:

sudo apt install pgbouncer

SSL/TLS Connection Errors

Add to connection string:

?sslmode=require

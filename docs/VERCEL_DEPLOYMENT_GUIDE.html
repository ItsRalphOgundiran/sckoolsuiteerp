1. Architecture Overview

Recommended Production Stack

ComponentServiceWhy This Choice
Frontend/BackendVercelServerless, auto-scaling, edge network
DatabaseSupabasePostgreSQL + Connection pooling + Row security
File StorageCloudinaryVercel Edge compatible, no local FS
Auth SessionsJWT (stateless)Required for serverless - no sticky sessions
CRITICAL for Vercel: Vercel uses serverless functions with ephemeral filesystems. You CANNOT use SQLite or local file storage. You MUST use PostgreSQL + external file storage.

3. Database Setup (Supabase)

1 Create Supabase Project
  1. Go to https://supabase.com/dashboard
  2. Click "New Project"
  3. Select your organization
  4. Project name: sckoolsuite-erp
  5. Database password: Generate strong password, SAVE IT!
  6. Region: Choose closest to your users (e.g., us-east-1, eu-west-1)
  7. Click "Create new project" (takes 2-3 minutes)
2 Get Connection Strings

Once project is ready:

  1. Click "Connect" (top right)
  2. Go to "ORMs" tab
  3. Select "Prisma"
  4. Copy BOTH connection strings:
# Transaction Pooler (for queries - connection pooled)
DATABASE_URL="postgresql://postgres.[project-ref]:[password]@aws-0-[region].pooler.supabase.com:6543/postgres?pgbouncer=true&connection_limit=1"

# Direct Connection (for migrations only)
DIRECT_URL="postgresql://postgres:[password]@db.[project-ref].supabase.co:5432/postgres"
IMPORTANT: connection_limit=1 is REQUIRED for Vercel serverless functions. Supabase free tier allows 60 concurrent connections. With pooling, this supports thousands of users.
3 Configure Database Extensions

In Supabase SQL Editor, run:

-- Enable required extensions
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
CREATE EXTENSION IF NOT EXISTS "pgcrypto";

4. Project Configuration

4 Verify Prisma Schema

Ensure prisma/schema.prisma has:

generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider  = "postgresql"
  url       = env("DATABASE_URL")
  directUrl = env("DIRECT_URL")
}
5 Configure Next.js for Vercel

Create/Update next.config.js:

/** @type {import('next').NextConfig} */
const nextConfig = {
  images: {
    remotePatterns: [
      {
        protocol: 'https',
        hostname: '**.supabase.co',
      },
    ],
  },
  // Ensure Prisma works on Vercel
  webpack: (config, { isServer }) => {
    if (isServer) {
      config.resolve.alias['@prisma/client'] = require.resolve('@prisma/client')
    }
    return config
  },
}

module.exports = nextConfig
6 Configure Package.json Build Script

Update package.json scripts:

"scripts": {
  "dev": "next dev",
  "build": "prisma generate && next build",
  "start": "next start",
  "lint": "next lint",
  "postinstall": "prisma generate"
}
prisma generate && next build ensures Prisma client is generated during Vercel build.

5. Environment Variables

7 Local Development (.env.local)

Create .env.local in project root (NEVER commit this file):

# ==========================================
# DATABASE (Supabase PostgreSQL)
# ==========================================
DATABASE_URL="postgresql://postgres.[ref]:[password]@aws-0-[region].pooler.supabase.com:6543/postgres?pgbouncer=true&connection_limit=1"
DIRECT_URL="postgresql://postgres:[password]@db.[ref].supabase.co:5432/postgres"

# ==========================================
# AUTHENTICATION (NextAuth.js v5)
# ==========================================
NEXTAUTH_URL="http://localhost:3000"
NEXTAUTH_SECRET="your-generated-secret-min-32-chars"

# ==========================================
# OPTIONAL: FILE UPLOADS
# ==========================================
# Cloudinary (recommended for Vercel)
CLOUDINARY_CLOUD_NAME="your-cloud-name"
CLOUDINARY_API_KEY="your-api-key"
CLOUDINARY_API_SECRET="your-api-secret"

# ==========================================
# APPLICATION
# ==========================================
NODE_ENV="development"
8 Generate NEXTAUTH_SECRET 
# Run this in terminal:
openssl rand -base64 32

# Or use Node.js:
node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"

Copy the output and use as NEXTAUTH_SECRET.

9 Test Local Build 
# Install dependencies
npm install

# Generate Prisma client
npx prisma generate

# Run migrations (creates database tables)
npx prisma migrate deploy

# Start development server
npm run dev

Visit http://localhost:3000/setup to verify everything works.

6. Vercel Deployment

10 Import Project to Vercel
  1. Go to https://vercel.com/dashboard
  2. Click "Add New..." → "Project"
  3. Import Git Repository: Select your SckoolSuite repo
  4. Click "Import"
11 Configure Build Settings

On the "Configure Project" page:

SettingValue
Framework PresetNext.js
Root Directory./ (leave default)
Build Commandprisma generate && next build
Output Directory.next (leave default)
Install Commandnpm install
12 Add Environment Variables (CRITICAL)

Expand "Environment Variables" section and add ALL of these:

Variable NameValue (from Supabase/Secrets)
DATABASE_URLYour Supabase transaction pooler URL
DIRECT_URLYour Supabase direct URL
NEXTAUTH_SECRETYour generated 32+ char secret
NEXTAUTH_URLhttps://your-project.vercel.app
NODE_ENVproduction
NEXTAUTH_URL must be your production domain!
For initial deploy, use the Vercel preview URL, then update after getting custom domain.
13 Deploy
  1. Click "Deploy"
  2. Wait for build (3-5 minutes)
  3. If build fails, check logs for errors
14 Run Database Migrations (One-time)

After first deploy, you must run migrations. Options:

Option A: Vercel CLI (Recommended)

# Install Vercel CLI
npm i -g vercel

# Login
vercel login

# Run migration with production database
vercel env pull .env.production
DATABASE_URL="your-production-url" npx prisma migrate deploy

Option B: Supabase SQL Editor

# Run in Supabase SQL Editor after first deploy:
# Check if tables exist
SELECT * FROM information_schema.tables WHERE table_schema = 'public';

If tables don't exist, you may need to run migrations locally pointing to production DB (be careful!).

7. Post-Deployment

15 Initial Setup
  1. Visit your deployed URL: https://your-project.vercel.app/setup
  2. Complete the 5-step setup wizard:
    • School Details
    • Academic Session
    • Academic Term
    • Admin User
    • Review & Activate
  3. After activation, you'll be redirected to login
  4. Login with the admin credentials you created

  1. In Vercel dashboard, go to your project
  2. Click "Settings" → "Domains"
  3. Add your domain (e.g., erp.yourschool.edu)
  4. Follow DNS instructions provided by Vercel
  5. Update NEXTAUTH_URL environment variable to your custom domain
  6. Redeploy the project

8. Troubleshooting

Build Fails with "Can't reach database server"

Cause: DATABASE_URL not accessible during build

Fix:

  • Verify IP allowlist in Supabase: Project Settings → Database → Network Restrictions
  • Add 0.0.0.0/0 (allow all) for serverless functions
  • Ensure connection string format is correct

Runtime Error: "PrismaClientInitializationError"

Cause: Prisma client not generated

Fix: Ensure prisma generate runs before build:

"build": "prisma generate && next build"

Authentication Not Working (Cookies Issue)

Cause: NEXTAUTH_URL mismatch or secure cookie settings

Fix:

  • Verify NEXTAUTH_URL matches your actual domain exactly
  • For custom domains, ensure HTTPS is configured
  • Check browser console for CORS errors

Database Connection Pool Exhausted

Cause: Too many concurrent connections

Fix:

  • Ensure connection_limit=1 in DATABASE_URL
  • Use Supabase Pooler (port 6543) not direct connection (port 5432)
  • Consider upgrading Supabase tier for more connections

9. Production Checklist

Before going live, verify ALL of these:
  • ☐ PostgreSQL database active (Supabase dashboard shows "Healthy")
  • ☐ All environment variables set in Vercel (check 5+ times)
  • ☐ NEXTAUTH_SECRET is 32+ random characters
  • ☐ NEXTAUTH_URL matches production domain exactly
  • ☐ Database migrations run successfully (tables exist)
  • ☐ Setup wizard completed and school activated
  • ☐ Admin user can login successfully
  • ☐ Custom domain configured (if applicable)
  • ☐ HTTPS working (no mixed content warnings)
  • ☐ Supabase backups enabled (Project Settings → Database)
  • ☐ Vercel monitoring enabled (Analytics)
